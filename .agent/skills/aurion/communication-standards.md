# 🏛️ Aurion Collective: Padrões de Comunicação (V1.0)

Para garantir a máxima fluidez e qualidade na produção do Coletivo Supremo, todos os agentes devem seguir estes protocolos de comunicação.

## 1. Protocolo de Handoff (Passagem de Bastão)
Sempre que um agente solicitar a ação de outro, a mensagem deve seguir esta estrutura:

- **Contexto**: O que já foi feito até agora.
- **Missão**: O que exatamente o próximo agente deve fazer.
- **Critérios de Sucesso**: Como saberemos que a tarefa foi bem executada.
- **Arquivos Relacionados**: Links para os arquivos que devem ser lidos ou modificados.

## 2. Loop de Reflexão (Self-Correction)
Antes de considerar uma tarefa concluída, o agente deve realizar uma "Auto-Crítica":
- "Este plano atende a todos os requisitos do usuário?"
- "Existe alguma falha de segurança ou performance óbvia?"
- "O próximo agente terá informações suficientes para agir sem me perguntar de novo?"

## 3. Feedback do Revisor (Quality Circle)
Ao rejeitar ou sugerir mudanças, o Revisor deve:
- Ser **específico** (indicar a linha ou o componente).
- Sugerir uma **alternativa** (não apenas dizer que está ruim).
- Validar se a mudança não quebra outras partes do sistema.

## 4. Uso da Memória (Knowledge Manager)
- Sempre consulte o `Knowledge Manager` para garantir que a terminologia e as decisões passadas sejam respeitadas em novas tarefas.
